# GSL Engine 🚀

**เอนจินการเพิ่มประสิทธิภาพด้วยฮิวริสติกขั้นสูงแบบกรรมสิทธิ์เฉพาะ สำหรับการแก้ปัญหาการจัดเส้นทางยานพาหนะ (VRP)**

GSL Engine เป็นแกนประมวลผลแบบปิด (Closed-source) ที่ออกแบบมาเพื่อแก้ปัญหาโลจิสติกส์ที่ซับซ้อน ด้วยประสิทธิภาพและความสามารถในการปรับตัวเชิงอัลกอริทึม  
Repository นี้จัดทำขึ้นเพื่อเป็น **แฟ้มสะสมผลงานด้านประสิทธิภาพ (Performance Portfolio)** โดยประกอบด้วยผลการทดสอบ (Benchmark), Log การทำงาน และหลักฐานเชิงประจักษ์ของความสามารถของระบบ

> ⚠️ **คำชี้แจง:** ไม่มีการเปิดเผยซอร์สโค้ดใน Repository นี้  
> โครงการนี้เปิดสำหรับ **ความร่วมมือด้านงานวิจัยและการประยุกต์ใช้งานในอุตสาหกรรม**

---

## 📌 สถาปัตยกรรมของเอนจินและรูปแบบปัญหาที่รองรับ

GSL Engine ถูกออกแบบในลักษณะ Modular เพื่อรองรับปัญหา VRP หลายประเภท:

- **CVRP** (Capacitated VRP)  
  *โมดูล:* `GSL_CVRP_V22` → ✅ ทดสอบและยืนยันผลแล้ว  

- **VRPTW** (VRP with Time Windows)  
  *โมดูล:* `GSL_VRPTW_V111` → ✅ ทดสอบและยืนยันผลแล้ว  

- **PDPTW** (Pickup & Delivery with Time Windows)  
  *สถานะ:* 🔄 อยู่ระหว่างพัฒนา (แกนหลักเสร็จแล้ว รอ Benchmark)

- **MDVRP** (Multi-Depot VRP)  
  *สถานะ:* 🔄 อยู่ระหว่างพัฒนา (Script ถูก Freeze แล้ว รอ Benchmark)

---

## ⚙️ สภาพแวดล้อมการประมวลผล

- แพลตฟอร์ม: Android (Mobile)
- Runtime: Python ผ่านแอป Pydroid 3  
- CPU: สถาปัตยกรรม Snapdragon  
- รูปแบบการทำงาน: Sequential / Batch Processing  
- การตรวจสอบคำตอบ: ตรวจสอบข้อจำกัด (Feasibility) แบบกำหนดแน่นอน (Deterministic)

> ระบบถูกออกแบบให้สามารถขยายไปทำงานบน Server หรือ High-Performance Environment ได้

### 🧠 นโยบาย Zero-Tuning

ใช้ **สคริปต์เดียว (Single unified script)** กับทุกชุดข้อมูล:

- ❌ ไม่มีการปรับแต่งพารามิเตอร์เฉพาะโจทย์  
- ❌ ไม่มีการ fine-tune ราย instance  
- ✅ ทำงานแบบอัตโนมัติทั้งหมด  

แนวทางนี้แสดงถึง **ความสามารถในการ generalize ของอัลกอริทึม**

## 🧠 กลยุทธ์ทางเทคนิค

GSL Engine ใช้แนวทาง heuristic เฉพาะทาง:

1. **Geometric Sensing**  
   การจัดกลุ่มเชิงพื้นที่เพื่อสร้างคำตอบเริ่มต้นคุณภาพสูง  

2. **Vehicle Minimization (K-Focus)**  
   ลดจำนวนรถก่อนปรับระยะทาง  

3. **Temporal Synchronization**  
   จัดการ Time Window อย่างมีประสิทธิภาพ  

4. **Universal Parsing Core**  
   รองรับหลาย format และตรวจจับค่า K อัตโนมัติ

## บริการและความร่วมมือทางธุรกิจ

เปิดรับการร่วมงานและให้คำปรึกษาระดับมืออาชีพ ในขอบเขตดังต่อไปนี้:

- **Logistics-as-a-Service (LaaS):** บริการระบบประมวลผลเส้นทางแบบเรียลไทม์สำหรับฟลีทรถขนส่งระดับองค์กร
- **High-Precision Modeling:** การออกแบบและพัฒนาอัลกอริทึมเฉพาะทาง สำหรับแก้ปัญหาซัพพลายเชนที่มีเงื่อนไข (Constraints) ซับซ้อนขั้นสูง
- **Technical Consultancy:** บริการให้คำปรึกษาทางเทคนิค, ทดสอบขีดจำกัดระบบเครือข่ายขนาดใหญ่ (Stress-testing) และการตรวจสอบประเมินประสิทธิภาพการจัดเส้นทาง (Optimization audits)
